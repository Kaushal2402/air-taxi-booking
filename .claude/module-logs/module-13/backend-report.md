## Key Implementation Notes

- `from __future__ import annotations` on all Python files using union syntax (Python 3.9 compliance)
- All endpoints auth-guarded with `_: AdminUser = Depends(get_current_admin_user)`
- Versioned publish flow: on publish, existing `live` rule for same zone+class (or route+category for air) is set to `past`, version incremented to max+1, rule_code regenerated with new version
- Road rule codes auto-generated as `PR-{ZONE_CODE}-{CLASS_CODE}-{VV}` (e.g. PR-BLR-SEDAN-01)
- Air rule codes auto-generated with UUID fragment (PR-AIR-{8 hex chars}) for uniqueness
- Fare simulator: handles base, distance, time, waiting charges; time-windowed modifiers (including overnight wrap e.g. 23:00–05:00); surge capped at surge_cap; toll pass-through; promo discount; returns full breakdown per rule
- `modifiers` JSON field handled defensively (works whether loaded as dict or object)
- `zone_name` and `vehicle_class_name` are computed and injected into `__dict__` at service layer (not stored in DB), satisfying RoadRuleResponse schema
